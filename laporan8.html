Laporan Praktikum Pemrograman Web: Migration, Seeding, Routing, Model, Controller, View

1. Pendahuluan

Praktikum ini bertujuan untuk memberikan pemahaman dan keterampilan fundamental kepada mahasiswa dalam membangun aplikasi web modern yang dinamis berbasis kerangka kerja PHP. Fokus utama pada modul ini adalah implementasi konsep arsitektur MVC (Model-View-Controller) secara utuh menggunakan Laravel. Pemahaman mengenai alur kerja kerangka kerja ini sangat penting karena Laravel menyediakan alat bawaan yang sangat terstruktur untuk menangani operasi basis data, rute aplikasi, dan penyajian antarmuka. Dengan mempraktikkan materi ini, mahasiswa diharapkan mampu membangun aplikasi operasi data dasar atau CRUD secara rapi dan mudah dipelihara pada tahap pengembangan lanjutan.

Tujuan Praktikum:
  • Mengelola versi struktur basis data menggunakan fitur Migration dan mengisi data awal menggunakan Seeding.
  • Mengatur pintu masuk aplikasi web menggunakan Routing serta membangun Model untuk interaksi langsung dengan basis data.
  • Mengembangkan Controller sebagai pusat logika atau jembatan penghubung antara Model dan View.
  • Membangun tampilan antarmuka halaman yang konsisten dan terstruktur menggunakan Blade Template Engine.

2. Langkah Kerja dan Implementasi

Berikut adalah penjelasan lengkap beserta tangkapan layar dari hasil eksekusi tahapan praktikum:

A. Konfigurasi Database

Pada tahap awal pembuatan aplikasi Laravel, konfigurasi basis data merupakan langkah yang sangat krusial agar aplikasi dapat berinteraksi dengan server lokal. Konfigurasi ini dilakukan dengan memodifikasi file environment yang berada di direktori utama proyek. Pada file tersebut, pengaturan koneksi basis data disesuaikan dengan menetapkan jenis koneksi ke MySQL dan mengubah nama basis data menjadi praktikum_laravel. Proses ini memastikan bahwa seluruh operasi manipulasi data ke depannya dapat terhubung secara lancar dengan server MySQL yang sedang berjalan pada perangkat lokal.

Gambar 2.1: Konfigurasi file environment untuk basis data
B. Migration dan Seeding

Migration di Laravel bertindak sebagai pengendali versi untuk basis data yang memungkinkan pengembang merancang, mengubah, atau menghapus struktur tabel hanya menggunakan sintaks PHP tanpa harus menulis kueri SQL secara manual. Melalui perintah artisan di terminal, sebuah kerangka migrasi dibuat untuk tabel produk lengkap dengan tipe data kolomnya. Setelah tabel berhasil dieksekusi, tahap selanjutnya adalah proses Seeding yang difungsikan untuk mengisi tabel tersebut dengan data pengujian awal secara otomatis. File seeder dikonfigurasi untuk menyisipkan beberapa entitas produk ke dalam basis data sehingga saat aplikasi diakses, data tersebut langsung tersedia untuk ditampilkan.

C. Pengaturan Routing dan Model

Routing merupakan sistem pengarah utama di mana setiap alamat URL yang diakses oleh pengguna akan dipetakan ke respon atau logika tertentu di dalam aplikasi. Pada praktikum ini, implementasi rute dikonfigurasi menggunakan teknik resource route untuk secara otomatis menyiapkan jalur khusus operasi CRUD. Bersamaan dengan itu, Model dibentuk untuk menangani komunikasi langsung aplikasi dengan basis data memanfaatkan fitur Eloquent ORM. Model ini secara cerdas merepresentasikan tabel produk di basis data dan dilengkapi dengan properti khusus agar manipulasi penambahan atau perubahan data dapat dilakukan dengan cepat dan aman.

Gambar 2.4: Konfigurasi Route dan struktur Model
D. Pembuatan Controller dan View

Controller berfungsi sebagai pengelola utama logika aplikasi yang bertugas menerima permintaan pengguna, memanggil data yang diperlukan dari Model, dan mengirimkannya ke View untuk ditampilkan. Pada tahap ini, controller dibuat untuk memiliki fungsi fungsi standar seperti menampilkan data, memproses pembaruan, hingga menghapus data. Sementara itu, antarmuka pengguna direpresentasikan melalui View yang dibangun menggunakan Blade Template Engine. Fitur penataan letak dari Blade ini memungkinkan pembuatan satu kerangka halaman utama yang dapat digunakan kembali secara dinamis oleh halaman lain, sehingga penulisan kode tampilan web menjadi jauh lebih efisien, modular, dan terstruktur.

3. Kesimpulan

Secara keseluruhan, praktikum ini membuktikan keandalan arsitektur MVC pada kerangka kerja Laravel dalam mempercepat siklus pengembangan aplikasi web. Pengelolaan skema basis data secara programmatic menggunakan Migration dan Seeding terbukti sangat memudahkan tim pengembang dalam membangun sistem. Selain itu, kolaborasi antara sistem Routing yang ringkas, Model Eloquent ORM yang cerdas, hingga fleksibilitas Blade Template Engine pada View menjadikan kode yang dihasilkan jauh lebih rapi, terorganisir, dan siap untuk dipelihara maupun ditingkatkan skalanya di masa mendatang.
